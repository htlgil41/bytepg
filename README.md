# 🐘bytepg🐘

Este repositorio es una investigacion y estudio sobre **Publication/Subscription** en PostgreSQL surgida por el vacio existencial de creer que soy parte del estandar **"Programador promedio"**

A continuacion se mostrara una breve introduccion de un foro en internet como texto principal: La replicación lógica de PostgreSQL permite replicar datos de forma selectiva entre bases de datos en tiempo real. A diferencia de la replicación física, que copia todo el clúster, la publicación/suscripción ofrece un control preciso sobre qué tablas se replican y dónde.

Los escenarios mas reales a nivel productivo son: Leer replicas de informes, Migracion de de bases de datos, Despliegues mutirregional, Intercambio selectivo de datos y por ultimo la Actualizacion de versiones. Prodrian existir mas casos al respecto pero nacen de una nececidad corporativa o una idea de proyecto pero en cuestion estods pueden ser un gran ejemplo.
